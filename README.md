# Utility Tools

A Node.js + Express project that serves a suite of free, privacy-friendly web-based utilities. All tools run entirely in your browser — the server just hosts static files and a tiny JSON API.

## SEO

- Each tool and content page has a unique `<title>`, meta description, keywords, canonical URL, Open Graph tags, and (where relevant) JSON-LD structured data.
- `/robots.txt` and a dynamic `/sitemap.xml` are generated by the Express server.
- Semantic HTML (`<header>`, `<main>`, `<article>`, `<footer>`) throughout.

## Quick start

```bash
cd utility-tools
npm install
npm start
```

Open <http://localhost:3000>.

Set `SITE_URL` and `PORT` as environment variables for production:

```bash
SITE_URL=https://yourdomain.com PORT=8080 npm start
```

The `SITE_URL` is used in the sitemap and robots.txt so links are absolute.

## Adding a new tool

1. Create `public/tools/<your-tool>.html` — use any existing tool as a template. Each page has:
   - A full SEO `<head>` (unique title, description, keywords, canonical, OG tags)
   - `<header class="topnav">` at the top
   - A `<main class="container">` with your tool UI
   - `<script src="/site.js"></script>` before `</body>`
2. Add the tool to the `TOOLS` array in `server.js`. It will automatically appear on the dashboard, be routed at `/<slug>`, and be included in the sitemap.

## Adding a new blog post

1. Create `public/pages/blog/<slug>.html` (model it on an existing post — full SEO `<head>` + `<article class="prose">` body).
2. Add the slug to the `urls` list in the `/sitemap.xml` handler in `server.js`.
3. Add a `<a class="blog-card">` entry to `public/pages/blog.html`.

## Notes

- Zero build step. No React/Vue/bundler. Just HTML, CSS, vanilla JS, and Express.
- All computation is client-side. Server never sees your values.
- A few tools use external CDNs (loaded once, no data uploaded):
  - QR Code Generator → `qrcode.js` from cdnjs
  - DOCX to PDF → `mammoth.js` from cdnjs
  - Markdown Preview → `marked.js` from cdnjs (with built-in fallback)
- The Spelling Practice tool uses the browser's built-in `speechSynthesis` API — no network required.
